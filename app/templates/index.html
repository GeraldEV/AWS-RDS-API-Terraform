Artists API Home

This page describes the endpoints of this Artists API!
It's not the prettiest but it gets the job done!
Note to self: This really should be a Swagger interface

Endpoints

GET /

This page!

GET /ping

Endpoint for reachability checks.
Response: HTML with body containing only "pong" and an HTTP 200 status code.

GET /health

Endpoint for health check. API confirms it can communicate with it's dependencies.
Response: JSON object body with the status of the API and an HTTP 200 status code.

GET /artists

All artists currently entered in the database.
Response: JSON List data for each artist with name and id.

POST /artists

Add a new artist into the database. Expects JSON body of form: {"artist": "<name>"}
Response: JSON object for the created entry and an HTTP 201 status code, JSON object for an already existing artist or JSON object describing error and an HTTP 400 status code otherwise.

GET /artists/<int: id>

Artist with ID matching <int: id>
Response: JSON object for the entry and an HTTP 200 status code or JSON object describing error and an HTTP 400 status code otherwise.

GET /artists/<int: id>

Update artist with ID matching <int: id>
Response: JSON object for the entry and an HTTP 200 status code or JSON object describing error and an HTTP 400 status code otherwise.

DELETE /artists/<int: id>

Delete artist with ID matching <int: id>
Response: HTTP 204 status code or JSON object describing error and an HTTP 400 status code otherwise.
